Nebula CSS Framework

Nebula.css is a work-in-progress CSS framework with the aim to quickly generate “building-block” classes for font sizes, colors, background colors and spacing. In addition to that, it also offers a nice typographic foundation + basic styles. It's inspired by BEM and OOCSS. It's a great starting kit for responsive websites, and allows you to easily scale type and spacing at different breakpoints.

The framework is built with SASS, and probably requires tweaking before compiling. That said, it can be used outside of a SASS pipeline together with other stylesheets.

Nebula loves spacing

For most block elements, Nebula provides no margins. This is so that the amount of spacing can be explicitly defined depending on the context and surrounding elements. In my experience, this leads to more consistent spacing across a page, with less chance of padding + margin adding up to inconsistent amounts compared to other elements on the page.

Nebula generates help classes

The top of the framework's styles.scss file contains a number of variables/maps you can set, which will result in the generation of building-block class names:

Colors & Background

$colors: (
  accent: #009CDA,
  …
);

.g-color-accent      { color: #009CDA; }
.g-background-accent { background-color: #009CDA; }

Borders

Border classnames are generated from two variables, first $bordersForColor defines what colors to generate borders for, and then $borderProperties defines what properties are styled:

$bordersForColor: 'accent', …
$borderProperties: 'border', 'border-bottom';

.g-border-accent        { border: 1px solid #009CDA; }
.g-border-bottom-accent { border-bottom: 1px solid #009CDA; }

Spacing

For every spacing value you define, the following classes get generated:

$spacingValue: (
  smallest: 5,
  …
);

.g-spacing-smallest  { margin: 5px;  }
.g-vspacing-smallest { margin-top: 5px;   margin-bottom: 5px;  }
.g-hspacing-smallest { margin-left: 5px;  margin-right: 5px;   }

.g-padding-smallest  { padding: 5px; }
.g-vpadding-smallest { padding-top: 5px;  padding-bottom: 5px; }
.g-hpadding-smallest { padding-left: 5px; padding-right: 5px;  }

As sometimes you might only want to give elements spacing on one side, any direction defined in $spacingDirections will generate additional directional classes for every $spacingValue defined:

$spacingDirections: 'top', 'bottom'; 

.g-spacing-top-smallest    { margin-top: 5px; }
.g-spacing-bottom-smallest { margin-bottom: 5px; }

.g-padding-top-smallest    { padding-top: 5px; }
.g-padding-bottom-smallest { padding-bottom: 5px; }

Font Sizes

Similarly, font size classes will be generated for $fontBaseSizes and $headerBaseSizes:

$fontBaseSizes: (
  normal: 16,
  …
);
$headerBaseSizes: (
  h1: 26,
  …
);

.g-font-normal { font-size: 16px; font-size: 1.6rem; }
h1, .g-font-h1 { font-size: 26px; font-size: 2.6rem; }

Since it's a header, we automatically create a h1 element, but we also create a class for this in case you want to style an element as a header, but do not want to associate it a header element's tag (for semantic reasons, for example).

Responsiveness

At the moment, Nebula does not have any elements which require “reassembly” on certain screen sizes. That said, the framework is built to scale to different sizes easily.

All sizes defined in the framework's configuration are regarded as base sizes that can be scaled for different media queries. Depending on the breakpoints your website requires, you define where you want font sizes or spacing to scale, and then call the media query to generate the rules again:

@media screen and (min-width:768px) {
  // Defining what size multiplier we use for the text and spacing for this breakpoint
  $bodyScale: 1.125;
  $headerScale: 1.8;
  $spacingScale: 2;
  $borderScale: 2;
  @include scaledSpacingAndType($spacingScale, $bodyScale, $headerScale);
  @include borderRules($borderScale);
}

This then generates the code necessary:

@media screen and (min-width: 768px) {
  .g-spacing-smallest { margin: 10px; }
  …
  body                { font-size: 18px; font-size: 1.8rem; }
  .g-font-normal      { font-size: 18px; font-size: 1.8rem; }
  h1, .g-font-h1      { font-size: 46.8px; font-size: 4.68rem; }
  …
  .g-border-accent     border: 2px solid #009CDA; 
}

The original values defined in Nebula's configuration have now multiplied by the scaling values defined above. For example, because the $spacingScale was set to 2, .g-spacing-smallest changed from 5 to 10.
